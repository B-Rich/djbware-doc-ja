argv0(1) - ucspi-tcp

名前

argv0 - 0 番目の引数を指定してプログラムを起動します。

書式

argv0 realname zero [arg ...]

解説

argv0 は、 ディスク上に realname という名前で置かれているプログラムを、 0 番目の引数として (realname の代わりに) zero を、 1 番目以降の引数として arg を指定して起動します。

応用

いくつかのプログラムは、 0 番目の引数に特別な意味を持たせています。 argv0 を利用すれば、 そのようなプログラムをシェルスクリプトから有効に起動することができます。

例えば次のように、 0 番目の引数に -csh を指定して /bin/csh を起動します。 

argv0 /bin/csh -csh

すると /bin/csh は、 引数に従いログインシェルとして動作します。

もう一つの例として、次のようなコマンドを見てください。 

tcpserver 0 ftp argv0 tcpd ftpd -l -A

このコマンドは、 次のような行を /etc/inetd.conf に記述した場合と同じ結果になります。 

ftp stream tcp nowait root tcpd ftpd -l -A

tcpd プログラムは、 0 番目の引数に ftpd 、 それ以降の引数に -l -A を指定して ftpd を起動します。 tcpd は tcpserver(1) に置き換えることができます。 次のように -x オプションを使用するとよいでしょう。 

tcpserver -x ftp.tcp.cdb 0 ftp ftpd -l -A
